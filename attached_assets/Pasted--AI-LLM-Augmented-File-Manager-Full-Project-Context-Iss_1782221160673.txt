# AI LLM Augmented File Manager - Full Project Context & Issues

## PROJECT OVERVIEW

**Purpose:** React Native app (Expo) for semantic file search with AI-powered action parsing and transaction-based file operations.

**Core Features:**
1. Semantic search by filename + file contents (embedded vectors, instant local results)
2. AI action parser: natural language → structured action plans
3. Transaction-based operations (all changes in tmp, approval required)
4. Snapshot & backup system (local + Google Drive abstraction)
5. .aiignore support (git-style nested ignore rules)
6. Sandboxed Node.js code execution for complex actions

**Tech Stack:**
- Frontend: React Native (Expo)
- Backend: Node.js
- Database: SQLite (local)
- Embeddings: Local embedding model (Xenova/transformers.js or similar)
- Code Execution: Sandboxed Node.js (vm2/isolated-vm)
- Storage: Local filesystem + Google Drive abstraction (not yet implemented)

## KEY ARCHITECTURAL NOTES

### Permission Handling (Critical for Expo)
- Use `expo-file-system` for file access (not direct fs)
- Use `expo-permissions` for requesting permissions
- On Android, request `READ_EXTERNAL_STORAGE` + `WRITE_EXTERNAL_STORAGE`
- Handle permission status: `GRANTED`, `DENIED`, `UNDETERMINED`
- Cache status to AsyncStorage to avoid repeated requests
- Graceful fallback if permissions denied (limit to app's document directory)

### Transaction Revert Flow
```
User clicks "Undo Transaction" 
  → System checks linked snapshot exists
  → Show confirmation with affected files list
  → If confirmed: restore files from snapshot OR execute reverse actions
  → Mark transaction as "reverted"
  → Update transaction log
  → Show success message
```

### .aiignore Parser
- Should read as TEXT file (not binary)
- Parse line by line
- Support comments (#) and empty lines
- Support negation patterns (!)
- Display in hierarchical tree (show nesting)
- Allow inline editing

### File Menu Requirements
- On React Native, use `expo-file-system` for file ops
- Menu should appear as ActionSheetIOS (iOS) or custom modal (Android/Web)
- Options: Copy, Move, Delete, Rename, Properties, Open
- Disable dangerous ops if transaction in progress
